📖 บทบรรยาย

สวัสดีครับ วันนี้จะมาพูดถึงเรื่อง "Agent Harness" ซึ่งเป็น concept ที่ hot มากในปี 2026 หลายคนที่ใช้ AI อาจคุ้นเคยกับ ChatGPT หรือ Claude ในรูปแบบแชท แต่จริงๆ แล้วสิ่งที่ทำให้ AI ทำงานได้จริงไม่ใช่แค่ Model เท่านั้น แต่เป็น "Harness" หรือโครงสร้างที่ล้อมรอบ Model นั้นไว้ วิดีโอนี้参考にจาก channel PannKs ที่อธิบายเรื่องนี้ได้ดีมากครับ

เริ่มจากพื้นฐานก่อน — LLM ตัวเดียว ≠ Agent ครับ เหมือนเวลาเรามีสมองแต่ไม่มีมือ ไม่มีเท้า — คิดได้แต่ทำอะไรไม่ได้ Raw LLM เป็นแค่โมเดลที่ตอบคำถามได้ แต่ไม่สามารถเปิดไฟล์, รันโค้ด, หรือจำสิ่งที่เราคุยเมื่อวานได้ แต่ถ้าเราใส่ "Harness" เข้าไป — เพิ่ม memory, เพิ่ม tool, เพิ่ม autonomous loop — LLM นั้นจะกลายเป็น Agent ที่ทำงานได้จริง เหมือนให้สมองมีมือ มีเท้า มีความจำ

นี่คือ Anatomy ของ Agent Harness ครับ — มี 4 ชั้นหลัก ชั้นในสุดคือ LLM Core ซึ่งเป็นส่วนที่ "คิด" ถัดออกมาคือ Memory System ที่จดจำ context, conversation history, ไฟล์ในโปรเจกต์ ถัดมาคือ Tool Integration ที่ให้ Agent ใช้งาน filesystem, terminal, browser ได้ และชั้นนอกสุดคือ Autonomous Loop ที่ทำให้ Agent ทำงานซ้ำๆ ได้เอง แก้ไขข้อผิดพลาด และดำเนินการต่อโดยไม่ต้องมีคนคุม

ลงรายละเอียด 4 องค์ประกอบหลัก ของ Harness — 1) Memory & Context: จัดการ context window, จำประวัติการคุย, index ไฟล์ในโปรเจกต์, อ่าน CLAUDE.md, รู้ git state — 2) Tool Use: อ่านเขียนไฟล์, รัน bash command, ค้นหาเว็บ, ควบคุม browser, เชื่อมต่อ MCP servers — 3) Error Handling: เมื่อผิดก็ retry, validate test, run hooks, graceful degradation, แก้ไขตัวเองได้ — 4) Autonomy: วางแผนหลายขั้นตอน, แบ่งงานย่อย, รัน background agents, ทำงาน session ยาวๆ ได้

Agent Harness ไม่ใช่ concept ทฤษฎีอย่างเดียว — มีใช้จริงแล้ว 4 ตัวหลัก ที่ทุกคนน่าจะคุ้นเคย — Claude Code จาก Anthropic เป็น terminal-based coding assistant ที่เรากำลังใช้อยู่ตอนนี้, OpenAI Codex จาก OpenAI เขียนด้วย Rust มี 75K+ GitHub stars, GitHub Copilot ที่มี agent mode ใน VS Code, และ Cursor ที่เป็น AI-first IDE ทั้ง 4 ตัวนี้มี harness ที่ต่างกัน แต่ concept เดียวกัน — คือทำให้ LLM ทำงาน autonomously ได้

มาดู Claude Code แบบละเอียดครับ — จุดเด่นคือ harness ที่ครบมาก มี CLAUDE.md เป็น persistent memory ที่เก็บ instructions ไว้ตลอด, tool use ที่ครบทั้ง file, bash, web, image, hooks ที่ automate pre/post actions ได้, MCP servers สำหรับเชื่อม tool ภายนอก, sub-agents ที่ทำงานคู่กันได้, และ plan mode ที่ออกแบบ architecture ก่อนเขียนโค้ด อย่างที่เราใช้เมื่อกี้ แค่สั่ง "สร้าง test1.txt มีเนื้อหา abcdef" Claude ก็อ่านเข้าใจ เขียนไฟล์ และ hooks ก็ auto-run ให้เลย

ฝั่ง OpenAI Codex มีจุดเด่นต่างออกไป — เขียนด้วย Rust เป็น open source จุดเด่นคือ sandbox execution รันโค้ดอย่างปลอดภัย, browser use เปิดเว็บ verify ผล, goals สำหรับวางแผนงานหลายขั้นตอน, และล่าสุดเมษายน 2026 สามารถควบคุม Mac desktop ได้แล้ว — คลิก, พิมพ์, รัน plugin ได้ 90+ ตัว ใช้ GPT-5.5 เป็น model หลัก และมี 75K+ GitHub stars แสดงว่า community ใหญ่พอสมควร

แล้ว ทำไม Harness ถึงสำคัญ? — จากกราฟนี้จะเห็นว่า Model อย่างเดียวได้ 75%, Harness อย่างเดียวได้ 90% (แม้แต่ model ธรรมดาถ้ามี harness ดีก็ทำงานได้เยอะ) แต่พอรวมกัน — Model ดี + Harness ดี — ได้ 98% ประโยคสำคัญจากวิดีโอคือ "ปัจจัยที่กำหนดความสามารถของ AI ไม่ใช่ความฉลาดของ model เท่านั้น แต่เป็น infrastructure ที่ล้อมรอบ model นั้น" นี่คือสิ่งที่เปลี่ยน mindset ของวงการ AI ในปี 2026

นี่คือ use cases จริง ที่ Agent Harness ทำได้ — สรุป Bug Reports: อ่าน bug reports จำนวนมาก สรุป pattern อัตโนมัติ, สร้าง Deploy Notes: อ่าน git log สร้าง release notes เอง, Incident Review: วิเคราะห์ logs สรุป root cause, เขียน Tests: อ่านโค้ดแล้วเขียน test ให้ครอบคลุม, Refactor Code: ปรับโครงสร้างโค้ด ลบ dead code, และ Web Scraping: ค้นหาเว็บ สรุปเนื้อดึงข้อมูลอัตโนมัติ ทั้งหมดนี้ทำได้โดยไม่ต้องมีคนคุมตลอด

สรุป — Agent Harness คือ infrastructure ที่ทำให้ AI กลายเป็น Agent ที่ทำงานได้จริง ไม่ใช่แค่ model ที่ "ฉลาด" แต่คือระบบที่ "ทำได้" ถ้าสนใจอยากดูเพิ่มเติม มีลิงก์ 3 ตัวด้านล่าง — วิดีโอต้นทางจาก PannKs, Anthropic Engineering Blog, และ LangChain Blog ที่อธิบาย anatomy ของ harness ไว้ละเอียดครับ ขอบคุณครับ มีคำถามไหม?
